GitHub-Befehle

git commit
Speichert Abbildung aller Dateien in diesem Projektverzeignis.
-> Copy & Paste.
Kopiert jedoch nur Änderungen, nicht das ganze Verzeichnis.
Git kann Commits als Menge von Änderungen zusammenfügen.
Jeder Commit wird protokolliert
-> ressourcenschonend, schneller Wechsel möglich
git branch <BRANCHNAME>
Verweise auf einen bestimmten Commit (Sicherung)
Braucht keinen Speicherplatz => viele machen
Bedeutet also "Ich möchte die Arbeit, die in diesem Commit und seinen Vorgängern steckt, sichern." 
git merge <BRANCHNAME>
Branches zusammenführen
=> Erzeugt neuen Commit aus mehreren Vorgänger-Commits
Bedeutet also "Ich möchte alle Arbeit von dem Vorgänger hier und dem dort
und allen ihren jeweiligen Vorgängern miteinander kombinieren" 
git log
Protokoll / Hashes anzeigen
Hash wird generiert, z.B. "fed2da64c0efc5293610bdd892f82a58e8cbc5d8"
Nur soviele nötig, bis es eindeutig ist, z.B. "git checkout fed2" reicht aus 
git commit ^
Einen Commit zurück gehen 
git commit ~5
Mehrere Commits zurück gehen durch Tilde (z.B. 5 Schritte) 
git cherry-pick
Commits auswählen und an aktuellen Arbeits-HEAD anhängen 
git reset
Änderungen zurücksetzen auf anderen Commit 
git revert
Änderungen rückgängig machen 
git rebase
Einarbeitung von Änderungen von einem Branch in einen anderen 
git fetch
Daten werden aus entferntem Repositiry heruntergeladen, aber nicht im Code aktualisiert 
git pull
Inhalte aus dem Remote-Repository werden herunterladen und direkt auf dem lokalen Repository aktualisiert
